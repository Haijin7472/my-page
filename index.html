Prototyping Stages

Showcasing the design evolution from low-fidelity sketches to vertical flow validation, clarifying the rationale for fidelity choices at each step.

1

Low-Fidelity Wireframes

In the initial stage, grayscale wireframes are used to quickly build the page skeleton. The focus is on the information architecture layout, such as placeholders for matrix calibration, data visualization, and coach list modules.

Fidelity Rationale

The core goal at this stage is "rapid iteration & structure validation". Eliminating visual distractions like color and typography forces designers and stakeholders to focus on content layout and functionality, reducing revision costs.

2

Horizontal Prototype (Global Navigation)

Building the global breadth structure. Implementing the bottom Tab Bar (Summary / Live Gym / Matrix / Coaches / Plan) to ensure seamless switching between 5 core views without diving into the specific functional logic of each page.

Fidelity Rationale

To test "breadth & information findability". Horizontal fidelity validates whether the 5-dimension structure aligns with user mental models, ensuring users never get lost without expending effort on deep logic for every subpage.

3

Vertical Prototype (Core Flow)

Deeply integrating the core business flow: "Calibrate Goal (Matrix) -> Generate Algorithm Recommendations -> Book Session (Plan & Coach)". This flow features high interaction fidelity and data linkage (e.g., selecting different body types auto-recalculates charts and schedules).

Fidelity Rationale

Using a high-depth, low-breadth strategy to validate "core value chain feasibility". Matrix algorithms and chart rendering are the product's core barriers, requiring near-code high-fidelity testing to ensure accurate data logic and smooth interactions.

Highly Reusable & Semantic Tailwind Architecture

After validating structure and flow, the project advances to maximum fidelity. By abstracting shared styles like .premium-card and refining Tailwind Theme configs (color system: Male #0A84FF / Female #FF375F, radius rules: 20px/24px), we ensure visual impact (glassmorphism, dark sci-fi style, transparent watermarks) while maintaining extreme component reusability for 100% high-fidelity code implementation.

5 Key UX Dimensions

A bottom-up panoramic deconstruction of the FITGRID app based on Jesse James Garrett's Elements of User Experience framework.

Dimension 01

Strategy
Strategy

  • User Needs: Escape generic fitness plans; require science-backed, personalized programs based on current physiology and clear goals (Body FITGRIDsition).
  • Business Goals: Build a digital loop: Goal Input → Algorithm Prescription → Coach Booking & Scheduling, achieving monetization and user retention.

Scope
Scope

Structure
Structure

Interaction Design & Information Architecture: Flat star navigation via bottom Tab Bar connecting 5 parallel core modules. Linear flows within functions, e.g.:

[Matrix View] → Select Gender → Set Baseline → Set Target → Trigger Validation / Generate Charts → [Guide] to Plan View

Modular Premium Cards separate functional zones. Optimized mobile touch targets (e.g., enlarged bottom nav). Information hierarchy uses size/color contrast: large bold headings for goals, high-saturation badges for status labels (e.g., "BOOKED").
